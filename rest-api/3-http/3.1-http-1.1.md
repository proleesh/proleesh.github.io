# 3.1 HTTP/1.1

REST API는 요청 메서드, 응답 코드, 메시지 헤더 등 [HTTP 버전 1.1](http://www.rfc-editor.org/rfc/rfc2616.txt) 의 모든 측면을 수용한다.

이 책에서는 두 장에 걸쳐 HTTP를 다룰 것인데, 3장에서는 요청 메서드와 응답 상태 코드에 대해 설명한다. REST API 설계에서 메타데이터를 다루는 방법은 HTTP 요청 및 응답 헤더에 대한 내용과 함께 4장에서 다룬다.

> NOTE\_ 3장의 예제 중 일부에서 사용하는 curl은 대부분은 개발 플랫폼에서 사용할 수 있는 명령라인 방식의 오픈 소스 웹 클라이언트다. REST API 관련된 일반적인 개발 작업에서는 curl을 쓰는 것이 더 편리하다. 특히 curl에서는 HTTP의 모든 기능을 사용할 수 있고 스크립트로 작업할  수도 있기 때문에, 개발자가 REST API를 테스트할 때 curl 명령으로 셸 스크립트 배치 파일을 작성하여 사용할 수 있어서 매우 편리하다.
