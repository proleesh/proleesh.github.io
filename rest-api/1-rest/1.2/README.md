# 1.2 웹 아키텍처

1993년 말에 공동으로 ['아파치 HTTP 서버 프로젝트'](http://httpd.apache.org)(Roy Fielding)은 웹 확장성에 관심이 있었다. 필딩은 웹 분석을 통해 확장성은 몇 가지 주요 제약점에 관심이 있었다. 필딩은 웹 분석을 통해 웹 확장성은 몇가지 주요 제약점에 따라 좌우된다는 것을 알게 되었고, 동료와 함께 실용적인 접근법으로 웹 구현 방법을 개선하기 시작했다. 그들의 목표는 제약점을 모두 만족시켜서, 웹이 지속적으로 확장하도록 하는 것이었다.

필딩은 제약점을 여섯 가지 범주로 묶어 웹의 구조적 스타일(Web's architectrual style) 이라고 불렀다. 언급한 웹의 구조적 스타일은 다음과 같다.
